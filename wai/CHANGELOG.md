# WAI — changelog

Notable changes to the WAI standard and the reference implementation.
Newest at the top. WAI is a draft standard; until v1.0 freezes, the
wire format may change and such changes are recorded here.

## v0.4.0 — current

### Neural condition: working end-to-end

- `tools/wai_encodec_encode.py` / `wai_encodec_decode.py` — first
  working **neural capability** (`wai.neural.encodec32`): source
  encodes audio via locally-installed EnCodec, ships only the
  quantized tokens in a standard WAI envelope (236 MB of model
  weights stay on the source's disk, never enter the envelope); sink
  with EnCodec installed decodes. Measured: **5 s of audio → 2.3 KB
  envelope (141× compressed)**, model on the wire = 0 B.
- Rust SDK gains `wai inspect` CLI: parses an envelope, resolves the
  capability + fallback against `sink_capabilities()`, and prints
  verdict (`DECODABLE via primary` / `via fallback` / `INERT here`).

### Browser runtime (`wai-web/`)

- New crate: wasm-bindgen wrapper exposing `parse(bytes) → WaiEnvelope`,
  `pack(manifest_json, payload)`, and `known_capabilities()`. 107 KB
  wasm + 16 KB JS glue.
- `wai-web/demo/wai.js` — thin JS shim with `WaiRuntime.register(cap,
  handler)` dispatch. Backends are **pluggable** (browser-native APIs,
  transformers.js, WebGPU compute, WebNN, ONNX runtime web) — WAI
  doesn't bundle any. The sink-side compute fabric is heterogeneous;
  the deployer picks per capability.
- `wai-web/demo/index.html` — drop-and-decode demo with handlers wired
  to: `createImageBitmap` for `wai.image.{png,jpeg,avif,jxl}`,
  `AudioContext` for `wai.audio.{opus,flac}`, placeholders for
  `wai.neural.*` showing where transformers.js / WebNN / WebGPU plug
  in. Run with `python3 wai-web/demo/serve.py`.
- 4 sample `.wai` files in `wai-web/demo/samples/` covering PNG, JXL,
  Opus, and EnCodec for testing.

### Spec: v1.1 multi-rendition envelope (SPEC.md §6.1)

- New magic `WAI2` for envelopes that carry multiple renditions of the
  same content (e.g. EnCodec + Opus, or AVIF + JPEG-XL). Reframed from
  "graceful fallback for missing capability" to **policy selection** —
  in a self-contained ecosystem every sink has every required
  capability; renditions exist so the sink picks by compute budget,
  bandwidth, or runtime preference (browser-native vs WebGPU vs WebNN
  vs native ML). v1.0 (`WAI1`) remains valid and unchanged.

## v0.3.0

**Pivot**: WAI is the **envelope + capability-dispatch + neural-shared-
prior** standard. The zeroth condition is a *menu of registered SOTA
codec capabilities* (AVIF, JPEG-XL, PNG, JPEG, Opus, FLAC, AV1, zstd,
XZ), not a custom codec. The Python prototype and all hand-rolled
codec implementations from v0.2.x were removed; Rust is the single
reference implementation and the SDK.

### Standard

- `SPEC.md` rewritten as a capability-menu spec. Mandatory floor:
  `wai.image.png`, `wai.audio.flac`, `wai.text.zstd` (every conforming
  sink MUST decode these). Recommended modern set: `wai.image.jxl`,
  `wai.image.avif`, `wai.image.jpeg`, `wai.audio.opus`, `wai.video.av1`,
  `wai.video.av1.lossless`, `wai.text.xz`. Neural capabilities
  (`wai.neural.*`) remain OPTIONAL and sink-advertised.
- Container envelope (`WAI1` magic + length-prefixed manifest + length-
  prefixed payload) and manifest schema are unchanged from v0.2.x.
- Conformance: bit-exact decode-equivalence with each capability's
  reference library (libjxl / libavif / libopus / libFLAC / rav1e / etc.).

### Rust reference (`wai-rs/`, crate `wai` 0.3.0)

- `cargo build --release` produces `libwai.dylib`/`.so`/`.dll` (cdylib),
  `libwai.a` (staticlib), and `libwai.rlib` (Rust-native). The crate
  type list is `["lib", "cdylib", "staticlib"]` — drop-in for any
  language's C-FFI binding.
- Codec wrappers: `image` (PNG, JPEG via `jpeg-encoder`, AVIF via
  `ravif` + `libavif-image` for decode, JPEG-XL via `jpegxl-rs`), `opus`
  (RFC 7845 Ogg-Opus container so output drops into ffmpeg/VLC/browsers),
  `claxon`+`flac-bound` for FLAC, `rav1e`+`dav1d` for AV1 (lossy +
  lossless), `zstd`+`xz2` for text.
- Generated C header: `include/wai.h` is regenerated by `build.rs` via
  `cbindgen` on every build. SDK consumers get a typed header rather
  than hand-writing `extern "C"` decls.
- Cross-platform linkage: `build.rs` probes `pkg-config` for libflac;
  the macOS-hardcoded `.cargo/config.toml` from v0.2.x is gone. Builds
  clean on macOS (Homebrew) and Linux (apt) without `RUSTFLAGS`.

### C FFI / SDK surface

All `wai_*` functions return a `wai_status_t`:

| code | meaning |
|---|---|
| `WAI_OK` (0) | success |
| `WAI_ERR_INVALID_ARG` (1) | null pointer / mismatched length / out-of-range dimension |
| `WAI_ERR_CODEC` (2) | codec-level failure (details in `wai_last_error()`) |
| `WAI_ERR_CONTAINER` (3) | bad magic / truncated envelope / unparseable manifest JSON |
| `WAI_ERR_PANIC` (4) | internal Rust panic caught at the boundary (a wrapper bug — never UB into the C caller) |

Every FFI function is wrapped in `std::panic::catch_unwind` so Rust
panics never unwind into C (UB). `av1_decode` validates header bounds
to prevent libdav1d's C-level `abort()`. `wai_last_error()` is per-
thread, populated on any non-OK return.

### Testing

- **13 lib tests**: per-codec round-trip + envelope round-trip
- **21 FFI robustness tests**: null pointers, length mismatches,
  dimension overflow, malformed magic, truncated payloads, bad JSON,
  invalid Opus sample rates, zero fps, garbage decoder input
- **10 fuzz smoke targets**: 1024 adversarial random byte sequences per
  decoder asserting no panic. Stable Rust, no `cargo-fuzz` toolchain
  needed. Found and fixed a real envelope-truncation panic.
- **`wai_bench` corpus walker**: quick mode (4 images + 4 audio + 3
  text + 2 video) for spot-checks; `--full` walks the entire corpus
  (169 images + 5 audio + 13 text + 12 video sequences = 588 codec
  runs in ~9.4 min). Per-codec aggregate stats: runs, ok / fail counts,
  WAI vs reference byte sizes, PSNR delta vs ffmpeg, encode time.
- **`--report file.json`** writes a structured bench report;
  `wai_bench_diff baseline.json head.json` flags regressions on bytes
  (±5% default tolerance), encode time (±25%), or new failures. Non-zero
  exit on regression — ready for CI gating.
- **GitHub Actions CI** (`.github/workflows/ci.yml`): ubuntu + macOS
  matrix, apt/brew codec deps, `cargo test --release`, cdylib/staticlib
  artifact verification, quick bench, JSON report uploaded as artifact.

### Removed

- The Python prototype (`wai/`) and Python BD-rate harness (`bench/`).
  v0.2.x's hand-rolled wire format is dead; Rust is the single source
  of truth.
- The `webm_lottie_new/wai-rs/` pre-pivot Rust crate (Wasm-decoder-in-
  bundle model). Replaced by the clean `wai-rs/` SOTA-wrapper crate.

## v0.2.0 — superseded (historical record)

The v0.2.x direction was hand-rolling custom codec wire formats (DCT +
quantization + rANS for image, MDCT for audio, I/P inter-frame for
video). Two byte-interoperable reference implementations (Python +
Rust) were built and cross-validated.

**Why superseded**: re-implementing JPEG/MPEG/Opus poorly is strictly
worse than wrapping the field's SOTA libraries. WAI's actual value is
the envelope + capability dispatch + the neural-shared-prior model,
which v0.3 keeps. Custom codecs are gone; standard libraries are in.

### What v0.2 had

- `SPEC.md` defined byte-precise wire formats for image (8×8 DCT-II +
  YCbCr + JPEG-Q + zigzag + DC-Δ + rANS), audio (orthonormal MDCT +
  sine window + TDAC), video (I/P inter-frame with integer-pel motion).
- Python (`wai/`) + Rust (`wai-rs/`) implementations, byte-equivalent on
  the lossless paths (rANS, container) and reconstruction-equivalent on
  the lossy paths.
- Honest measured standing: image at JPEG-1992 parity (~5% behind
  libjpeg), audio uniform-MDCT (not perceptually competitive with MP3),
  video I/P at ~41% smaller than MJPEG floor but far behind H.264.

### Licensing (unchanged)

- Apache-2.0 for the standard and the reference implementation.
  Royalty-free implementation is a precondition for adoption (VP9/AV1
  open-codec precedent; the H.264/H.265 royalty trap is avoided).
